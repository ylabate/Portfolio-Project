Sécurité & Bonnes Pratiques

Protocoles implémentés dans l'architecture pour sécuriser les données et les transactions :

Authentification JWT à Double Niveau

Utilisation de jetons d'accès (Access tokens) à durée de vie courte et de jetons de rafraîchissement (Refresh tokens) stockés de manière sécurisée pour renouveler les sessions de façon invisible.

Cryptage & Hachage Bcrypt

Les mots de passe ne sont jamais stockés en clair. Ils subissent un hachage unidirectionnel hautement sécurisé via Flask-Bcrypt lors de l'inscription et de la vérification à la connexion.

Sécurisation des Webhooks Stripe

Le point de terminaison de validation de paiement (Webhook) vérifie la signature de la requête reçue avec la clé secrète Stripe pour bloquer toute tentative d'injection frauduleuse de commandes.

Protection Contre les Injections SQL

Le backend utilise l'ORM SQLAlchemy. Les requêtes sont systématiquement préparées et paramétrées, ce qui neutralise complètement les failles classiques d'injections de scripts SQL.

Démarrage Rapide

Lancez l'environnement complet de développement (Frontend + Backend + Base de données) en local :

Configuration en Local

Cloner et lancer l'application en local via Docker Compose en quelques commandes bash :

Terminal Bash
git clone https://github.com/ylabate/Portfolio-Project.git
cd Portfolio-Project/levelup
echo "STRIPE_API_KEY=sk_test_..." > .env
echo -e "SECRET_KEY=dev\nJWT_SECRET_KEY=dev" > backend/.env
make dev

Lancer dans le Cloud
